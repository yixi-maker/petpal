# PetPal 内测使用指南

> 欢迎参加 PetPal 内测！以下是一些基本操作指南和注意事项。
> 内测期间所有功能均为测试版本，如有问题请通过反馈渠道联系我们。

---

## 如何注册

1. 打开测试地址：**https://staging.petpal.example.com**（或内测群内公布的最新地址）
2. 在登录页面输入手机号（任意 11 位数字即可，如 `13800000001`）
3. 点击「获取验证码」，验证码输入 **123456**（内测期间验证码固定为此值）
4. 阅读并同意《用户协议》和《隐私政策》，勾选同意框
5. 点击「登录」按钮，系统将自动为你创建账号并登录

> 提示：内测期间不校验真实手机号，也不会发送真实短信。如需测试多个角色，可以使用不同手机号创建多个账号。

## 主要功能

### 创建宠物档案
- 登录后点击底部「+」或进入「我的」页面
- 填写宠物名字、选择种类（狗狗/猫猫等）、品种、性别、体型
- 可上传宠物头像照片
- 创建后可随时编辑宠物信息

### 发布动态
- 首页点击右下角「发布动态」按钮（橙色 + 图标）
- 输入文字内容，可以添加图片
- 选择发布时关联的宠物（可选）
- 发布后动态会出现在首页信息流中
- 支持对动态进行点赞和评论

### 发现附近宠物
- 点击底部「发现」标签
- 查看附近其他宠物用户的动态
- 可以关注感兴趣的宠物
- 按城市筛选内容（北京 / 上海 / 深圳等）

### 地图找宠物友好地点
- 点击底部「地图」标签
- 查看城市中的宠物友好地点（宠物医院、宠物公园、宠物友好咖啡馆等）
- 点击地点卡片查看详情
- 内测期间地图为示意地图，地点数据为占位数据

### AI 健康助手
- 点击底部「健康」标签
- 选择需要咨询的宠物
- 填写症状描述、持续时间、食欲/饮水/精神状况
- 点击「开始 AI 健康咨询」获取分诊建议
- **AI 建议仅供参考，不能替代专业兽医诊断。如宠物出现紧急状况，请立即前往就近宠物医院。**

### 私信好友
- 在「我的」页面点击「私信」
- 可以与已关注的宠物主人私信聊天
- 支持文字消息

### 管理后台
- 管理员登录入口：`/admin/login`
- 可管理用户、内容审核、举报处理
- 此功能仅面向内测运营团队

### 反馈模板

提交 Bug 报告时，建议包含以下信息：

```
【问题类型】Bug / 功能建议 / 体验问题
【严重程度】阻塞使用 / 影响使用 / 小问题 / 建议
【页面/功能】在哪里遇到的问题（如：发布动态页）
【操作步骤】1. 打开xxx页面 2. 点击xxx 3. 输入xxx
【预期结果】期望发生什么
【实际结果】实际发生了什么
【设备信息】手机型号 + 浏览器 + 系统版本
【截图】如有请附上截图
```

## 已知问题

以下是内测期间已知的限制和问题：

| 问题 | 说明 | 影响 |
|------|------|------|
| 验证码为测试模式 | 验证码固定为 `123456`，不发送真实短信 | 无法测试短信到达率 |
| 地图为示意地图 | 地图区域显示占位内容，非真实高德地图 | 无法测试定位和导航功能 |
| AI 建议仅供参考 | AI 分诊使用 mock 数据，不调用真实 AI API | 分诊结果不代表真实诊断 |
| 图片存储为本地 | 上传的图片保存在服务器本地，非云存储 | 重启/迁移可能导致图片丢失 |
| 未配置 HTTPS | 内测环境可能使用 HTTP 协议 | 建议不要在公共 WiFi 下使用 |
| PWA 不可用 | Service Worker 已禁用，不支持离线缓存 | 无法添加到主屏幕离线使用 |

## 内测注意事项

1. **测试数据**：内测期间产生的所有数据（宠物、动态、图片等）可能会在测试结束后清除，请勿上传个人隐私信息。
2. **账号安全**：内测期间手机号无需真实验证，请勿使用真实个人手机号进行测试。
3. **功能范围**：内测仅验证核心社区功能（注册、发布、互动、宠物档案），第三方服务集成（短信、地图、AI、云存储）不在本次测试范围。
4. **崩溃反馈**：如遇到页面白屏或崩溃，请截图并记录操作步骤，通过反馈渠道提交。
5. **多设备测试**：欢迎在不同设备（iOS Safari、Android Chrome）上测试移动端适配效果。
6. **反馈时效**：一般问题我们会在 1-2 天内回复，紧急问题（核心功能不可用）会在 2 小时内响应。
